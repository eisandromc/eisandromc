# Matriz de ativação global

## Sempre ativos
- investigador-de-codigo
- arquiteto-de-software
- implementador
- testes
- revisor-tecnico

## Ativar conforme cenário
- refatorador, quando houver dívida técnica
- seguranca, quando houver auth, API pública, segredos ou dados sensíveis
- devops e github-actions, quando houver pipeline, deploy ou infra
- observabilidade e performance, quando houver problemas de produção ou gargalo
- documentador-tecnico, quando a mudança exigir atualização pública de docs

## Ativar por stack
Use somente os agentes das stacks existentes no repositório. Exemplo:
- SaaS web moderno, typescript, nodejs, react, nextjs, postgresql, docker, github-actions
- ERP corporativo, csharp, dotnet, aspnet-core, sql, azure
- Mobile cross-platform, dart, flutter, firebase
- API enterprise, java, spring, postgresql, kubernetes, aws
